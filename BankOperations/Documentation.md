# Документация на проекта: Банкови операции

## 1. Описание на техническото решение

### 1.1 Структура на проекта
Проектът е разработен с ASP.NET Core Web API за бекенд (.NET 10). Структурата на проекта следва N-Layer архитектура:
- **Controllers**: Съдържат API ендпойнти за обработка на HTTP заявки (Clients, BankAccounts, Credits).
- **Services**: Съдържат бизнес логиката (CreditService за изчисляване на погасителни планове).
- **Data (Repository)**: `ApplicationDbContext`, който се грижи за връзката с базата данни (Entity Framework Core).
- **Models**: Дефинират структурата на данните (Client, PhysicalPerson, LegalEntity, BankAccount, Credit, Installment).
- **Handlers**: Глобално прихващане на грешки (`GlobalExceptionHandler`).

### 1.2 Схема на базата данни (Entity Framework Core)
Базата данни е релационна и се управлява от `ApplicationDbContext`. Тя съдържа следните таблици (DbSets):
- **Clients**: Базов модел за всички клиенти (Съдържа общи данни).
  - **PhysicalPersons**: Наследява Clients (За физически лица - Име, Фамилия, ЕГН).
  - **LegalEntities**: Наследява Clients (За юридически лица - Име на фирма, ЕИК, Представител).
- **BankAccounts**: Банкови сметки(IBAN, Наличност, Статус, Притежател).
- **Credits**: Кредити (Вид, Сума, Срок). Всеки кредит е обвързан с клиент.
- **Installments**: Вноски по кредити (Размер, Главница, Лихва, Остатък).

Приложени са `HasPrecision(18,2)` конфигурации за всички десетични (decimal) полета като суми и наличности, за да се гарантира точността на финансовите данни.

### 1.3 Ключови компоненти от логиката
- **Анюитетно изчисляване на кредита**: Логиката за генериране на погасителен план е реализирана в `CreditService`. Използва математическа формула за анюитетна вноска, където месечната вноска е фиксирана, но съотношението главница/лихва се променя с времето.
- **Table-Per-Hierarchy (TPH) / Table-Per-Type (TPT)**: Обектно-ориентирано наследяване за клиентите. Физическите и юридическите лица се съхраняват ефективно използвайки EF Core механизмите за наследяване.
- **Тестове**: Разработени са Unit Tests за бизнес логиката (`CreditServiceTests`) и контролерите, гарантиращи правилната работа на изчисленията и управлението на акаунти. Проектът с тестовете: `BankOperations.Test`.

### 1.4 Използвани технологии
- **Backend framework**: ASP.NET Core Web API (.NET 10)
- **ORM / Database**: Entity Framework Core, SQL Server
- **Unit Testing**: xUni
